## 2. Presentación del proyecto para entrevista

El proyecto principal que articula esta presentación es un **proyecto intermodular de floristería**, planteado como una aplicación orientada a gestionar productos, compras y relaciones básicas entre distintas entidades del sistema.

Desde una perspectiva de entrevista, este proyecto demuestra capacidad para trabajar sobre una necesidad concreta y transformarla en una solución técnica estructurada, conectando interfaz de consola, servicios, acceso a datos y base de datos relacional.

### Qué problema resuelve

El proyecto busca organizar la gestión interna de una floristería, permitiendo trabajar con flores, clientes, trabajadores, compras y ventas dentro de una misma aplicación.

Este enfoque permite demostrar comprensión de procesos reales de negocio, validación de datos, gestión del stock y relaciones entre tablas y objetos del dominio.

### Tecnologías utilizadas

Las principales tecnologías y conceptos que se reflejan en el proyecto son los siguientes:

- **Java**, como lenguaje principal para la lógica de aplicación y la organización por capas.
- **MySQL**, para el almacenamiento persistente de los datos del sistema.
- **SQL**, para definir y consultar las estructuras y relaciones de la base de datos.
- **JDBC**, como mecanismo de conexión entre la aplicación Java y la base de datos.
- **Git y GitHub**, para el control de versiones, la publicación del proyecto y la presentación profesional del trabajo.[1]

### Qué demuestra este proyecto

Visto desde empleabilidad, este proyecto no solo enseña código, sino también criterio técnico: separación entre entidades, DAOs, servicios y menús; validación de entradas; control de errores; y evolución progresiva del diseño a medida que aparecen nuevas necesidades.

También permite explicar durante una entrevista cómo se ha pensado una solución de extremo a extremo, desde el modelo de datos hasta la interacción con el usuario y la persistencia final.

## 4. Reflexión personal

Este proyecto me deja un aprendizaje especialmente útil porque permite pasar de la teoría a una implementación completa en un caso cercano a un entorno real.

### Qué he aprendido

Entre los aprendizajes más relevantes destaco la organización del código por capas, la conexión entre Java y MySQL, el uso de DAOs y servicios, la validación de datos de entrada y la importancia de mantener una estructura clara conforme el proyecto crece.

También refuerza mi capacidad de detectar errores, refactorizar código, mejorar la experiencia de uso y pensar no solo en “que funcione”, sino en “que esté bien diseñado”.

### Qué me ha costado más

La parte ha sido la coordinación entre base de datos y lógica de aplicación, especialmente al trabajar con relaciones entre entidades, validaciones, stock, transacciones y control de errores.

Además, al avanzar en el desarrollo, también he necesitado revisar imports, tipos, estructura de clases y consistencia entre servicios, DAOs y modelo, algo muy habitual en proyectos en crecimiento.

### Que mejoraría

Como mejora futura, el proyecto podría evolucionar hacia una presentación más visual, una mejor documentación pública en GitHub, una organización todavía más modular y una futura adaptación hacia una interfaz web que permite interactuar con la BBDD.

También sería una buena mejora incorporar más validaciones de negocio, mejor presentación del portfolio y una explicación más clara del proyecto.
